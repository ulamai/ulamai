# v0.2.1 Execution Plan (In Progress)

Target: tighten inference-time proving quality while preserving reproducibility and benchmark comparability.

## Priority Order

1. Planner/replan loop in search.
2. Diagnostics-guided self-correction in `prove --prove-mode llm` and formalize.
3. Archon-style inference profiles (`generate -> rank -> verify`) under fixed budget.
4. Parallel Lean execution plumbing for larger search fanout.
5. Regression updates for parity checks on new behavior.

## Milestones

### M1: Planner/Replan + Subgoal Memory

Scope:
- Add lightweight state-level tactic memory cache.
- Reuse cached tactics before fresh LLM suggestions.
- Detect stagnation and inject explicit replan hints into prompts.

Acceptance:
- Search logs show planner cache hits and stagnation-triggered replan events.
- No regressions in current smoke flows (`mock` + `dojo`).

Status:
- Started in `ulam/search/best_first.py`:
  - planner cache memory
  - stagnation-triggered replan hinting
  - scripted + best-first integration
- Added planner counters to trace metadata and bench reports:
  - trace `.meta.json`: `search_stats.*`
  - bench summary/json/markdown: planner totals + per-case counters
- Extended `bench-compare` with planner deltas:
  - `planner_replan_triggers_total`
  - `planner_cached_tactic_tries_total`

### M2: Stronger Correction Loop

Scope:
- Promote diagnostic summaries to first-class repair context.
- Track repeated failure clusters and force strategy pivots.

Acceptance:
- Fewer repeated identical failures in LLM mode under fixed budgets.

Status:
- Started in `ulam/cli.py` (`run_prove_llm`):
  - failure-cluster counting (`_llm_error_cluster`)
  - repeated-cluster strategy pivot hints (`_llm_strategy_pivot_error`)

### M3: Inference Profiles

Scope:
- Add profile presets for budgeted candidate generation, ranking, and verification passes.
- Keep deterministic metadata for benchmark comparability.

Acceptance:
- Profile settings are visible in report metadata.
- `bench-compare` can compare profile runs cleanly.

Status:
- Implemented in `prove` + `bench`:
  - `--inference-profile {default,balanced,explore,verify}`
  - `--gen-k`, `--exec-k`, `--verify-level`
- Search path now enforces generate/rank/verify budgets per state:
  - request budget (`generation_budget_per_state`)
  - strict/none verification filters before execution
  - execution cap after ranking (`execution_budget_per_state`)
- Trace + bench metadata include profile fields.
- `bench-compare` labels/markdown now show inference profile and budgets.

### M4: Parallel Execution Substrate

Scope:
- Add optional worker pool abstraction for tactic/repair evaluation.
- Preserve replay-safe trace output ordering.

Acceptance:
- Same solved result as serial path on smoke suite.
- Reduced wall-clock time on multi-case runs.

### M5: Regression + Reporting

Scope:
- Expand fixed-suite usage (`regression100` plus curated slices).
- Add planner/replan counters to benchmark summary.

Acceptance:
- New counters appear in JSON + Markdown reports.
- Gate thresholds remain enforceable.

Status:
- Added planner-aware and profile-aware parity defaults in `bench-compare --gate`:
  - planner thresholds:
    - `max_planner_replan_triggers_increase`
    - `max_planner_cached_tactic_tries_drop`
  - comparability checks (strict by default in gate mode):
    - same suite SHA256
    - same inference profile/budget signature
  - explicit escape hatches:
    - `--allow-suite-mismatch`
    - `--allow-profile-mismatch`
- Wired release-script gate defaults in `scripts/run_bench_campaign.sh`:
  - `--compare-to <baseline_report.json>` runs `bench-compare --gate`
  - emits `compare.json`/`compare.md` artifacts per run

## Immediate Next Tasks

2. Run profile smoke comparisons (`default` vs `balanced`/`verify`) on internal suite.
3. Publish and pin the first canonical baseline report JSON used by `--compare-to` release gates.
